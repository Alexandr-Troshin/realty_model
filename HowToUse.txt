HowToUse
1. Обновление данных о торгах с roseltorg
    1.1 С сайта roseltorg.ru скачиваем в эксель данные по торгам с фильтрами:
        -   текст поиска - "жилого, УГС,  центр управления, реновации"
        -   ИНН организатора - 7704515009
        -   сортировка - "Дата окончания подачи заявок" по убыванию
    1.2 Полученный файл пересохраняем в формате xlsx в файл procedures.xlsx в папку procedure_folder.
    1.3 Запускаем приложение с пунктом "1 - Обновить данные по торгам roseltorg"
    * Обновится файл bidding.csv, содержащий данные с 01.01.2023г. по торгам roseltorg.
        Дубликаты по id процедуры удаляются с сохранением последних данных.

2. Получение данных по активным торгам
    2.1 Запустить приложение с пунктом "2 - актуальные лоты"
    2.2 При работе приложения могут появляться уточнения по адресам, отсутствующими в базе ЖКХ.
        выводится название адреса из строки (как оно прописано в лоте) и вариант, как распознала система.
        Если система распознала правильно - необходимо нажать y.
        Усли распознано неправильно (как правило, может отсутствовать название улицы) - можно попробовать
        внести в файл conv_dictionary.xls строку, которая заменит часть адресной строки на соответствующую
        запись во второй колонке (например, ул.Любимова (без пробела) может не распознаться как улица.
        Необходимо в первую строку вписать исходный вариант, во вторую - с пробелом "ул. Любимова"
        и повторно нажать 'y').
    2.3 После распознавания адресов запускается проверка наличия данных о метро и этажности дома по каждому
        адресу. Проверка выполняется по наличию записей. При отстутствии записи о метро и этажности -
        Парсятся данные об адресе с сайта flatinfo.ru. Первично приложение пытается отправить распознанный
        адрес сайту. При наличии схожих адресов - открывается страница дома, с которой парсится адрес.
        Если полученный адрес после распознавания совпадает с введенным - парсятся данные о доме.
        Если не совпадает - выдается запись "адреса не совпадают" и предложение ввести правильный url.
        Если на сайте flatinfo можно найти соответствующий дом (ввести чуть по-другому адрес) -
        необходимо подтвердить введение нового url (набрать 'y') и скопировать url дома из
        адресной строки браузера.
        После этого также может выдать запрос на несовпадение адреса - если уверены в правильности нового
        адреса - необходимо еще раз ввести 'y' и повторно вставить url (запишется принудительно).
        Затем все новые адреса проверяются через сайт 2gis.ru - если данные об этажности и году постройки
        совпадают (+/- 1 год) - данные о доме сохраняются в gkh_base.csv.
        Если данные не совпадают - на экран выводятся по каждому объекту сохраненные данные и данные с
        2gis. В файле new_buildings.xlsx выводятся записи по несовпавшим адресам. Здесь можно вручную
        откорректировать необходимые данные о доме - они проверятся еще раз и сохранятся принудительно
        при повторном подтверждении. После добавления данных в gkh_base.csv  файл new_buildings.xlsx
        будет очищен.
        * Дополнительно, если постфактум обнаруженый неправильные данные - можно в gkh_base.csv очистить
        данные в записи (строго контролировать количество запятых) и оставить только адрес и ссылку на
        павилную страницу flatinfo - тогда система не будет пытаться искать адрес, а соберет данные с
        указанного url.
        Данные сохраняются в файл realty_model_actual_lot_data.csv (файл перезаписывается полностью).
    2.4 После сбора данных по все лотам приложение попробует соединить датафрейм с данными по торгам
        bidding.csv. Соединение происходит по номеру процедуры в конце url сайта roseltorg (если
        ссылка начинается с 178fz - в качестве ключа в базе торгов используется id процедуры).
        Данные лотов (с добавление количества заявок в поле 'participant_qty' из данных roseltorg)
        сохраняются в папке output c названием '[YY]_[mm]_[dd]_realty_model_actual_lot_data.csv'

    * Запустится парсинг investmoscow.ru с параметрами отбора актуальных лотов. Обновляются файлы
        gkh_base.csv (если есть новые адреса)

3. Получение данных по прошедшим торгам
    3.1 Запустить приложение с пунктом "3 - прошедшие лоты"
    3.2 Перед парсингом приложение выдаст запрос номера страницы для парсинга (как правило, достаточно
        отпарсить 1 страницу - более 100 лотов).
        В процессе работы при наличии неизвестных адресов возможно уточнение данных (см. п.п. 2.2, 2.3).
    3.3 Данные сохраняются в файл realty_model_historical_lot_data.csv (в файл добаялются новые данные).
    3.4 После сбора данных по все лотам приложение попробует соединить датафрейм с данными по торгам
        bidding.csv. Соединение происходит по номеру процедуры в конце url сайта roseltorg (если
        ссылка начинается с 178fz - в качестве ключа в базе торгов используется id процедуры).
        Данные лотов (с добавление количества заявок в поле 'participant_qty', данных о цене победителя
         'win_price' и наименовании победителя 'win_name' из данных roseltorg)
        сохраняются в папке output c названием '[YY]_[mm]_[dd]_realty_model_historical_lot_data.csv'
